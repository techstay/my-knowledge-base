## Git 提交规范

### 提交格式

```
<emoji> <类型>: <简短描述>
```

### 类型说明

| 类型       | Emoji | 说明                       |
| ---------- | ----- | -------------------------- |
| `feat`     | ✨    | 新增功能                   |
| `fix`      | 🐛    | 修复问题                   |
| `docs`     | 📝    | 文档更新                   |
| `style`    | 💄    | 格式调整（不影响代码运行） |
| `refactor` | ♻️    | 代码重构                   |
| `chore`    | 📦    | 构建/工具/配置相关         |
| `perf`     | ⚡    | 性能优化                   |

### 示例

```
📝 docs: 更新拳皇 13 新手教程，统一方向符号为 emoji
🐛 fix: 修复防御取消 GC 缩写不明的问题
✨ feat: 添加 markdownlint 配置文件
♻️ refactor: 重构教程结构，删除重复内容
📦 chore: 初始化 git 仓库，添加 .gitignore
```

### 提交信息规范

- 保持简短，一到两行
- 第一行概括主要变更
- 如有需要，空一行后添加详细说明
- 使用中文描述

### 操作边界

- 严格按照用户说明进行最小操作，不要擅自推送
- 用户只要求提交时，只执行 `git commit`，不执行 `git push`

## Shell 命令使用原则

**优先使用 Python 脚本，避免直接使用 Shell 命令。**

| 场景                           | 推荐方式                            | 原因                   |
| ------------------------------ | ----------------------------------- | ---------------------- |
| 文件操作（重命名、移动、复制） | `python -c "import os/shutil; ..."` | 跨平台，路径处理统一   |
| 批量文件处理                   | Python 脚本                         | 避免 cmd/pwsh 语法差异 |
| 文本内容替换                   | Python 脚本                         | 编码处理更可靠         |
| 简单目录查看                   | `dir` / `ls`                        | 可以直接使用           |

### 为什么优先用 Python？

1. **跨平台一致**：Windows/cmd、Windows/pwsh、macOS/bash 语法不同，容易失败
2. **路径处理可靠**：中文路径、空格、特殊字符在 Python 中处理更稳定
3. **编码统一**：Python 默认 UTF-8，避免 cmd 的 GBK 编码问题
4. **错误率低**：Shell 命令需要频繁转义，Python 字符串更直观

### 示例对比

❌ **Shell 命令（容易失败）**：

```bash
# Windows cmd 语法，Linux/macOS 不兼容
for %f in (*.png) do magick "%f" "output/%~nf.webp"
```

✅ **Python 脚本（跨平台）**：

```python
import os
from pathlib import Path

for f in Path(".").glob("*.png"):
    # 处理逻辑
    pass
```

## 其他规范

- 执行任何修改操作前，先读取/检查当前状态
- 批量操作前先小范围测试
- 遇到错误时，优先分析原因，而不是重复尝试相同命令

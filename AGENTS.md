# Git 提交规范

## 提交格式

```
<emoji> <类型>: <简短描述>
```

## 类型说明

| 类型 | Emoji | 说明 |
|------|-------|------|
| `feat` | ✨ | 新增功能 |
| `fix` | 🐛 | 修复问题 |
| `docs` | 📝 | 文档更新 |
| `style` | 💄 | 格式调整（不影响代码运行） |
| `refactor` | ♻️ | 代码重构 |
| `chore` | 📦 | 构建/工具/配置相关 |
| `perf` | ⚡ | 性能优化 |

## 示例

```
📝 docs: 更新拳皇 13 新手教程，统一方向符号为 emoji
🐛 fix: 修复防御取消 GC 缩写不明的问题
✨ feat: 添加 markdownlint 配置文件
♻️ refactor: 重构教程结构，删除重复内容
📦 chore: 初始化 git 仓库，添加.gitignore
```

## 提交信息规范

- 保持简短，一到两行
- 第一行概括主要变更
- 如有需要，空一行后添加详细说明
- 使用中文描述
